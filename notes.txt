A real-time database is broadly defined as a data store
designed to collect, process, and/or enrich an incoming series 
of data points (i.e., a data stream) in real time, typically immediately 
after the data is created.

 ->why bearer with token for authorizarion
 Long before bearer authorization, this header was used for Basic authentication. For interoperability, the use of these headers is governed by W3C norms, so even if you're reading and writing the header, you should follow them. Bearer distinguishes the type of Authorization you're using, so it's important.
 Bearer Tokens are the predominant type of access token used with OAuth 2.0. A Bearer Token is an opaque string, not intended to have any meaning to clients using it.
